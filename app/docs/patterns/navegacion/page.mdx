# Navegación

La navegación responde a una pregunta simple: **¿dónde estoy y a dónde puedo ir?**
Cumbre organiza la navegación en **tres niveles** y cada uno tiene un patrón
recomendado. Mezclarlos o duplicarlos es la forma más rápida de perder a tu usuario.

## Tres niveles de navegación

| Nivel | Qué responde | Patrones |
| --- | --- | --- |
| **Primaria** | "¿En qué parte del producto estoy?" | Header sticky, sidebar persistente |
| **Secundaria** | "¿En qué sección dentro de esta parte?" | Tabs, subnav horizontal |
| **Terciaria** | "¿En qué paso / subsección fina?" | Breadcrumbs, in-page anchors, pagination |

<Callout variant="info" title="Regla de 3">
  Si necesitás **más de 3 niveles** de navegación visibles al mismo tiempo,
  la información está mal agrupada. Primero revisá la arquitectura; después el patrón.
</Callout>

## Navegación primaria

### Header sticky

Ideal para productos **horizontales** (pocos destinos, alta frecuencia de uso).

<Preview label="Header sticky" background="muted" align="start" minHeight={120}>
  <nav
    aria-label="Principal"
    className="flex w-full min-w-fit items-center justify-between gap-6 rounded-md border border-border bg-surface px-4 py-3 sm:px-6"
  >
    <span className="text-sm font-semibold tracking-tight text-text">Cumbre</span>
    <ul className="flex items-center gap-3 text-sm text-text-secondary sm:gap-6">
      <li><a href="#" aria-current="page" className="font-medium text-text">Inicio</a></li>
      <li><a href="#" className="hover:text-text">Productos</a></li>
      <li><a href="#" className="hover:text-text">Equipo</a></li>
      <li><a href="#" className="hover:text-text">Contacto</a></li>
    </ul>
  </nav>
</Preview>

**Consideraciones:**

- `aria-current="page"` en el link activo — lectores de pantalla lo anuncian.
- Mantené el header **fijo al scroll** solo si es útil (navegación transversal constante).
  Un header sticky en un formulario largo es ruido.
- En móvil, colapsá a un menú hamburguesa accesible (`aria-expanded`, focus trap en el
  drawer, `Escape` para cerrar).

## Navegación secundaria

### Tabs

Para alternar entre **vistas paralelas** de un mismo contexto (p. ej. "Detalle", "Historial",
"Comentarios" de un paciente).

<Preview label="Tabs" background="muted" align="start" minHeight={140}>
  <div
    role="tablist"
    aria-label="Secciones del paciente"
    className="flex w-full min-w-fit gap-4 border-b border-border sm:gap-6"
  >
    {[
      { label: "Detalle", active: true },
      { label: "Historial", active: false },
      { label: "Turnos", active: false },
      { label: "Documentos", active: false },
    ].map(({ label, active }) => (
      <button
        key={label}
        role="tab"
        aria-selected={active}
        className={
          active
            ? "-mb-px border-b-2 border-primary-800 px-1 py-2 text-sm font-medium text-text dark:border-accent-500"
            : "-mb-px border-b-2 border-transparent px-1 py-2 text-sm text-text-secondary hover:text-text"
        }
      >
        {label}
      </button>
    ))}
  </div>
</Preview>

**Consideraciones:**

- `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls` — obligatorios.
- Navegación con teclado: **flechas izquierda/derecha** entre tabs, `Home`/`End` para
  primera/última.
- No más de **5–6 tabs** visibles. Si necesitás más, es navegación primaria
  disfrazada.

### Subnav horizontal

Similar a tabs pero para **navegación real entre páginas**, no entre views de un componente.
Visualmente casi idéntico a tabs, semánticamente es `<nav>` con links.

## Navegación terciaria

### Breadcrumbs

Para productos con **jerarquía profunda** (admin panels, CMS, expedientes). Muestran dónde
estás y permiten subir un nivel.

<Preview label="Breadcrumbs" background="muted" align="start" minHeight={80}>
  <nav aria-label="Migas de pan" className="w-full">
    <ol className="flex items-center gap-2 text-sm text-text-secondary">
      <li><a href="#" className="hover:text-text">Pacientes</a></li>
      <li aria-hidden className="text-text-tertiary">/</li>
      <li><a href="#" className="hover:text-text">García, María</a></li>
      <li aria-hidden className="text-text-tertiary">/</li>
      <li aria-current="page" className="font-medium text-text">Historial clínico</li>
    </ol>
  </nav>
</Preview>

**Consideraciones:**

- `<nav aria-label="Migas de pan">` envolvente — el label ayuda a navegar entre landmarks.
- `aria-current="page"` en el último item. El último item **no es link** (ya estás ahí).
- El separador es decorativo: `aria-hidden` y CSS, no texto real en el DOM de ítems.

### Pagination

Para listas largas paginadas. Cumbre prefiere **infinite scroll con trigger explícito**
("cargar más") para listas de contenido y **pagination numerada** para tablas de datos
serias.

<Preview label="Pagination numerada" background="muted" align="start" minHeight={100}>
  <nav aria-label="Paginación" className="flex w-full min-w-fit items-center justify-between gap-2">
    <button className="shrink-0 rounded-md border border-border-strong px-3 py-1.5 text-sm text-text-secondary hover:bg-surface-muted">← Anterior</button>
    <ul className="flex items-center gap-1 text-sm">
      <li className="hidden sm:block"><a href="#" className="rounded-md px-3 py-1.5 text-text-secondary hover:bg-surface-muted">1</a></li>
      <li><a href="#" aria-current="page" className="rounded-md bg-primary-800 px-3 py-1.5 font-medium text-text-on-primary dark:bg-accent-500 dark:text-primary-950">2</a></li>
      <li><a href="#" className="rounded-md px-3 py-1.5 text-text-secondary hover:bg-surface-muted">3</a></li>
      <li aria-hidden className="px-2 text-text-tertiary">…</li>
      <li className="hidden sm:block"><a href="#" className="rounded-md px-3 py-1.5 text-text-secondary hover:bg-surface-muted">12</a></li>
    </ul>
    <button className="shrink-0 rounded-md border border-border-strong px-3 py-1.5 text-sm text-text-secondary hover:bg-surface-muted">Siguiente →</button>
  </nav>
</Preview>

## Consideraciones de accesibilidad

- Todo menú interactivo debe ser **navegable con teclado**: `Tab` para entrar/salir,
  flechas para moverse dentro, `Enter`/`Space` para activar.
- Estado activo **siempre** con `aria-current="page"` — no alcanza con color.
- Menús desplegables: `aria-expanded`, `aria-haspopup`, focus trap cuando el menu está
  abierto, `Escape` para cerrar.
- En dropdowns y combobox, `role="listbox"` + `aria-activedescendant` según WAI-ARIA APG.
- Skip-links visibles al focus: el primer tab de la página debería ser "Saltar al
  contenido".
- Contraste AA en estado normal, hover, focus y selected — en ambos temas.

## Cuándo usar / Cuándo NO usar

<Callout variant="do">
  - Elegí **un patrón por nivel** y mantenelo en todo el producto. Consistencia > elegancia.
  - En productos largos, combiná **sidebar (primaria) + tabs (secundaria) + breadcrumbs
    (terciaria)** si hace falta.
  - Resaltá el estado activo con **más de un señalizador** (color + peso + borde).
</Callout>

<Callout variant="dont">
  - No tengas **dos navegaciones primarias** (header + sidebar con los mismos items).
    Elegí una.
  - No uses tabs para navegar entre **páginas reales** con URLs propias — usá links
    de verdad con subnav semántico.
  - No escondas la navegación detrás de hovers o animaciones sofisticadas. Cada click que
    pedís de más, lo pagás en abandono.
</Callout>
