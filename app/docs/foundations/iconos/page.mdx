# Iconos

Cumbre usa **[Tabler Icons](https://tabler.io/icons)** como única librería de iconografía —
específicamente el paquete `@tabler/icons-react`. Cumbre lo adoptó por tres razones: **geometría
precisa** (alineada con el lenguaje visual del sistema), **catálogo amplio** (más de 5.000 iconos
con licencia MIT) y **variantes filled + outline** (cubre los dos usos del sistema).

## Estilo por default: Filled

Por coherencia con la estética del sistema, **el sistema usa la variante Filled en todos los
casos posibles**. Los iconos filled tienen peso visual consistente con la tipografía en peso
medium y no compiten con el texto circundante.

### La única excepción: glyphs direccionales

Los iconos direccionales (flechas) no tienen variante Filled en Tabler — son glyphs line-based
por naturaleza. Para estos, Cumbre usa el outline. Es la **única excepción aceptada** en el sistema.

  </div>
</Preview>

## Uso

En cualquier componente de la librería:

```tsx
import { IconCheckFilled, IconMailFilled } from "@tabler/icons-react";

<IconCheckFilled size={16} className="text-success-500" />
<IconMailFilled size={16} className="text-text-secondary" />
```

Los iconos heredan el color del texto contenedor vía `currentColor`, así que podés cambiar color
con la utility `text-*` de Tailwind sin tocar el SVG.

## Tamaños recomendados

Tres tamaños cubren el 99% de los casos. Evitá tamaños arbitrarios — la precisión visual se
sostiene con una escala limitada.

<Callout variant="info" title="Regla de oro">
  Los iconos nunca llevan color por sí solos. **El color lo define el contexto** (estado,
  jerarquía, ubicación). Un icono no tiene que ser reconocible por su color — tiene que
  ser reconocible por su forma.
</Callout>

## Naming convention

Tabler Icons usa el prefijo `Icon` y el sufijo `Filled` para la variante filled:

| Variante | Patrón | Ejemplo |
| --- | --- | --- |
| Filled (default Cumbre) | `Icon<Name>Filled` | `IconCheckFilled`, `IconMailFilled` |
| Outline (solo glyphs direccionales) | `Icon<Name>` | `IconArrowUpRight`, `IconArrowLeft` |

Al buscar en [tabler.io/icons](https://tabler.io/icons), filtrá por **Filled** primero.
Si no existe la variante filled para lo que necesitás, revisá si hay una alternativa con forma
antes de recurrir al outline.

## Cuándo usar iconos

<Callout variant="do" title="Cuándo usar">
  - Para **reforzar** una acción textual (botón "Guardar" con icono de check opcional).
  - Como **única señal** en controles con tooltip explícito (icon buttons con `aria-label`).
  - En **badges de estado** donde el color + forma comunican más que el texto solo.
  - En **empty states** y **hero sections** como elemento visual focal.
  - En **menús de navegación** densos donde el icono ayuda a escanear.
</Callout>

<Callout variant="dont" title="Cuándo NO usar">
  - Como **decoración** vacía al costado de cada h2 o bullet — agrega ruido sin función.
  - Para **reemplazar texto crítico** en acciones destructivas o flujos legales — los iconos
    solos son ambiguos, WCAG recomienda siempre texto acompañante en contextos sensibles.
  - Para **transmitir información compleja** — un icono no puede decir "3 expedientes pendientes
    de aprobación desde el martes". Usá texto.
  - **Más de uno junto al otro** sin separación semántica clara — el lector no sabe qué es qué.
</Callout>

## Accesibilidad

- **Iconos decorativos**: agregar `aria-hidden` para que lectores de pantalla los ignoren.
  El texto circundante ya comunica la acción.
- **Iconos funcionales** (icon buttons sin texto): el botón contenedor debe tener
  `aria-label` con la acción descrita. El icono adentro va con `aria-hidden`.
- **Nunca** poner el icono en un `alt` o dejar que el lector lo interprete — los iconos no
  tienen texto alternativo semánticamente útil.

```tsx
// Decorativo — el texto ya explica
<Button>
  <IconCheckFilled aria-hidden /> Guardar
</Button>

// Funcional — solo icono, el botón debe tener aria-label
<button aria-label="Cerrar modal">
  <IconCircleXFilled aria-hidden />
</button>
```

## Instalación en un proyecto consumer

Los componentes de Cumbre que usan iconos (Button loading, Checkbox, Callout, ThemeToggle)
ya traen el import. Si querés usar iconos adicionales en tu propio código:

```bash
npm install @tabler/icons-react
```

Ya está listado como `dependency` de `@andesia/cumbre`, pero podés instalarlo explícito en tu
proyecto para que tu bundler tenga visibilidad directa del paquete.
