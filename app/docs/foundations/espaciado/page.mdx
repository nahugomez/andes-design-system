# Espaciado

Todo el sistema respira en una grilla de **4px**. No es arbitrario: ese múltiplo encaja con
el renderizado subpixel de las pantallas modernas y te deja componer layouts predecibles
sin pelear con decimales.

## Base · 4px

Cada paso de la escala es un múltiplo de 4. Esto aplica a `padding`, `margin`, `gap`, `space-*`.
Si estás usando un valor que no es múltiplo de 4, probablemente estás **resolviendo mal** un
problema de alineación.

<Callout variant="info" title="Regla de 4">
  Si un diseño "pide" 7px o 13px, casi siempre lo que pide es 8px o 12px. Confiá en la escala.
</Callout>

## Ritmo vertical

Las páginas de Cumbre usan **tres ritmos** claros:

- **Dentro de un componente** (`2` a `4`, 8–16px) — padding, gaps de controles.
- **Entre componentes relacionados** (`6` a `10`, 24–40px) — cards en un grid, items en una lista.
- **Entre secciones de página** (`12` a `20`, 48–80px) — cambio de tema, separación de landing.

<Callout variant="do">
  - Elegí un **step fijo por nivel** y mantenelo en toda la vista.
  - Duplicá el ritmo cuando subís de nivel: si dentro de un bloque usás `4`, entre bloques usá `8` o `10`.
  - Preferí `gap` en containers flex/grid antes que márgenes en hijos — evitás collapsing y hace el layout más predecible.
</Callout>

<Callout variant="dont">
  - No mezcles steps adyacentes (`5` y `6`) en la misma vista: el ojo los lee como error de alineación.
  - No uses `margin-top` para separar bloques si el container puede hacer `gap`.
  - No metas valores arbitrarios (`17px`, `23px`) "porque queda mejor": estás rompiendo el grid.
</Callout>

## Padding de containers comunes

| Contexto | Step | Valor |
| --- | --- | --- |
| Botón sm | `3` horizontal, `0` vertical (altura fija) | 12px / 32px |
| Botón md | `4` horizontal | 16px / 40px |
| Input md | `3` horizontal | 12px / 40px |
| Card sm | `4` all | 16px |
| Card md | `6` all | 24px |
| Card lg | `8` all | 32px |
| Page container (móvil) | `4` horizontal | 16px |
| Page container (desktop) | `8` a `12` horizontal | 32–48px |

## Gap vs padding

- **Padding** define el aire **adentro** de un contenedor.
- **Gap** define el aire **entre hijos**.

Si el padding del container y el gap de los hijos pelean, gana el gap. Mantenelos en la
misma familia (p. ej. padding `6`, gap `4`).

## Accesibilidad

- Tocables mínimos: **44 × 44 px** (altura + padding combinados).
- Gap mínimo entre controles tocables adyacentes: **8px** para evitar taps accidentales.
- En lectura lineal, respetar ritmo visual ayuda a usuarios con dificultades cognitivas a
  mapear jerarquía sin releer.

## Cuándo usar / Cuándo NO usar

<Callout variant="do">
  - Usá steps **pares** (`2`, `4`, `6`, `8`) para ritmo consistente.
  - Subí al siguiente múltiplo (4 → 8 → 16 → 32) cuando cambiás de nivel de jerarquía.
</Callout>

<Callout variant="dont">
  - No uses `1.5` o `2.5` de forma habitual: son excepciones, no norma.
  - No compenses mal espaciado con bordes o colores. Si algo respira mal, arregla el espacio.
</Callout>
