# Cumbre

Cumbre es la base sobre la que construimos los productos de AndesIA.
Una colección de **fundaciones visuales**, **componentes** y **patrones** pensados
para producir interfaces coherentes, accesibles y consistentes entre productos.

## El problema que resuelve

Sin un sistema compartido, cada producto vuelve a decidir colores, espacios y patrones desde
cero. Los usuarios ven productos que se ven y se sienten distintos aunque vengan del mismo
equipo. Cumbre centraliza tres cosas:

- **Tokens** — colores, tipografía, espacios, sombras, movimiento. Una sola fuente de verdad.
- **Componentes** — piezas probadas, accesibles (WCAG 2.1 AA) y consistentes con los tokens del sistema.
- **Patrones** — cuándo usar qué, y cómo combinarlo.

## Estado del sistema

Cumbre está en **v0.1 · desarrollo activo**. Lo que está documentado es estable;
lo que falta está pendiente. Esta página crece con el sistema.

## Versionado

Seguimos [Semantic Versioning](https://semver.org/lang/es/). Los tokens y las APIs
públicas de componentes son contrato. Los cambios incompatibles requieren un incremento
mayor de versión, con notas de migración.
