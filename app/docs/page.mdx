# Cumbre · _Design System_ de AndesIA

Cumbre es la base sobre la que construimos los productos de AndesIA.
Una colección de **fundaciones visuales**, **componentes** y **patrones** pensados
para producir interfaces coherentes, accesibles y con carácter propio.

---

## ¿Por qué un Design System?

Porque sin un sistema compartido, cada producto inventa su propia rueda. Los usuarios
pagan el costo: inconsistencia, fricción, desconfianza. Cumbre resuelve eso centralizando:

- **Tokens** — colores, tipografía, espacios, sombras, movimiento. Una sola fuente de verdad.
- **Componentes** — piezas probadas, accesibles (WCAG 2.1 AA) y temáticamente coherentes.
- **Patrones** — cuándo usar qué, y cómo combinarlo. El cómo importa tanto como el qué.

## Filosofía

> Diseñamos para que el producto desaparezca y quede la tarea.
> Cumbre se nota cuando algo se rompe — nunca cuando funciona.

Seguimos [Semantic Versioning](https://semver.org/lang/es/). Los tokens y las APIs
públicas de componentes son contrato. Los cambios breaking requieren major bump y
se comunican con migration notes.
