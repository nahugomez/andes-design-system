# Checkbox

El checkbox es para decisiones **independientes entre sí**: varias opciones, cualquier
combinación es válida. Si las opciones son mutuamente excluyentes, lo que necesitás es un
radio — no un checkbox.

## Ejemplo básico

<Preview align="start" minHeight={100}>
  <div className="flex items-center gap-2">
    <Checkbox id="basic-checkbox" defaultChecked />
    <Label htmlFor="basic-checkbox">Acepto los términos y condiciones</Label>
  </div>
</Preview>

La asociación con `<Label htmlFor>` hace que **clickear el label marque el checkbox** — es
UX básica y a11y obligatoria.

### Indeterminate · cuándo

El estado indeterminate se usa para el **padre** de un grupo de checkboxes cuando **algunos
hijos están marcados y otros no**. Es la convención universal para "select all" parcial.

```tsx
const allChecked = items.every((i) => i.selected);
const someChecked = items.some((i) => i.selected);
const state = allChecked ? true : someChecked ? "indeterminate" : false;

<Checkbox checked={state} onCheckedChange={toggleAll} />;
```

semánticamente, y los lectores de pantalla la anuncian como un grupo.

## Props

| Prop | Tipo | Default | Descripción |
| --- | --- | --- | --- |
| `checked` | `boolean \| "indeterminate"` | — | Estado controlado. |
| `defaultChecked` | `boolean` | `false` | Estado inicial (no controlado). |
| `onCheckedChange` | `(checked: boolean \| "indeterminate") => void` | — | Callback al cambiar. |
| `size` | `"sm" \| "md"` | `"md"` | Tamaño del checkbox y del ícono. |
| `disabled` | `boolean` | `false` | Deshabilita el control. |
| `required` | `boolean` | `false` | Marca el control como requerido. |
| `name` | `string` | — | Nombre para form submission. |
| `value` | `string` | `"on"` | Valor en form submission. |
| `className` | `string` | — | Clases adicionales. |

Basado en [Radix Checkbox](https://www.radix-ui.com/primitives/docs/components/checkbox) —
acepta todos los props de `CheckboxPrimitive.Root`.

## Accesibilidad

- Implementado sobre Radix Checkbox — cumple WAI-ARIA APG.
- Navegación por teclado: `Tab` para enfocar, `Space` para toggle.
- Estados expuestos vía `data-state="checked" | "unchecked" | "indeterminate"`.
- `focus-visible:shadow-focus` para foco visible (SC 2.4.7).
- Asociación con label via `htmlFor` — clickear el label cambia el estado.
- Contraste AA en border, fondo y el check dentro (cumple en light y dark).

## Cuándo usar / Cuándo NO usar

<Callout variant="do">
  - Usá checkbox cuando las opciones son **independientes** entre sí.
  - Usá indeterminate para el padre de un grupo con selección parcial.
  - Agrupá con `<fieldset>` + `<legend>` cuando hay un título común.
</Callout>

<Callout variant="dont">
  - No uses checkbox para opciones **excluyentes** — eso es radio.
  - No uses checkbox para "activar / desactivar" una funcionalidad instantánea — eso es
    switch/toggle.
  - No dejes checkboxes sin label visible "para ahorrar espacio". Un label oculto
    (`sr-only`) es el mínimo aceptable.
</Callout>
