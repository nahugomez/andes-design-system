# Layout

El layout es la **arquitectura silenciosa** de una interfaz. Se nota cuando falla y
desaparece cuando funciona. Cumbre usa **grillas predecibles**, **containers
acotados** y **un puñado de patrones** que resuelven el 90% de los casos.

## Grilla · responsive · 4 / 8 / 12 columnas

La grilla base de Cumbre colapsa según el dispositivo:

| Breakpoint | Columnas | Gutter |
| --- | --- | --- |
| Mobile (`< md`) | 4 | 16px |
| Tablet (`md` a `lg`) | 8 | 24px |
| Desktop (`lg +`) | 12 | 24px |

12 es divisible por 2, 3, 4 y 6 — por eso es versátil en desktop. 8 en tablet mantiene la
lógica de mitades y cuartos. 4 en mobile fuerza decisiones duras: si no entra en 4, no entra.

<Preview label="Grilla responsive · 4 / 8 / 12 columnas" background="muted" minHeight={140}>
  <div className="grid w-full grid-cols-4 gap-4 md:grid-cols-8 md:gap-6 lg:grid-cols-12">
    {Array.from({ length: 12 }).map((_, i) => (
      <div
        key={i}
        className={`${i < 4 ? "flex" : i < 8 ? "hidden md:flex" : "hidden lg:flex"} h-16 items-center justify-center rounded-sm border border-border bg-surface text-2xs font-mono text-text-tertiary`}
      >
        {i + 1}
      </div>
    ))}
  </div>
</Preview>

## Container widths

Toda página vive dentro de un container. Cumbre usa dos tamaños estándar:

| Contexto | Max width | Clase |
| --- | --- | --- |
| App / dashboard | 1536px | `max-w-screen-2xl mx-auto` |
| Contenido / docs | 1200px | `max-w-6xl mx-auto` |
| Formulario / detalle | 640px | `max-w-2xl mx-auto` |
| Lectura | 65ch | `max-w-prose mx-auto` |

<Callout variant="info" title="Padding horizontal">
  Acompañá el container con `px-4 sm:px-6 md:px-8 lg:px-12`. Ver
  [Breakpoints](/docs/fundaciones/breakpoints) para la escala completa.
</Callout>

## Grid vs flex · cuándo cada uno

Regla simple:

- **Grid**: layout **bidimensional** (filas × columnas). Cuando hay que alinear en los dos ejes.
- **Flex**: layout **unidimensional**. Cuando hay que distribuir en una sola dirección.

No uses grid "porque es moderno". Un navbar horizontal es flex; una lista vertical es flex.
Una galería de cards con múltiples filas es grid.

<Callout variant="do">
  - Usá `grid` cuando necesitás que los items se alineen **en columnas** aunque las filas
    tengan distinto alto.
  - Usá `flex` cuando necesitás **distribuir espacio** (space-between, grow, shrink) en una
    dirección.
  - Usá `gap` en ambos; evitá márgenes en los hijos.
</Callout>

## Patrones recurrentes

### Stack · apilado vertical

El más común. Todo en columna, con ritmo vertical consistente.

    <div className="rounded-md border border-border bg-surface p-4 text-sm text-text">Acción</div>
  </div>
</Preview>

### Sidebar · contenido principal + panel lateral

Sidebar fijo (lg+) o off-canvas (móvil). El sidebar toma un **ancho fijo** (256–320px),
el main toma el resto.

<Preview label="Sidebar layout" background="muted" align="start" minHeight={260}>
  <div className="flex w-full flex-col gap-4 md:flex-row md:gap-6">
    <aside className="rounded-md border border-border bg-surface p-4 text-sm text-text md:w-48 md:shrink-0">
      Sidebar
      <div className="mt-2 flex flex-col gap-1 text-xs text-text-secondary">
        <span>Item 1</span>
        <span>Item 2</span>
        <span>Item 3</span>
      </div>
    </aside>
    <main className="rounded-md border border-border bg-surface p-4 text-sm text-text md:flex-1">
      Main content
    </main>
  </div>
</Preview>

### Split · dos paneles equivalentes

Detail views, comparisons, master-detail. Ambos paneles son jerárquicamente pares.

    <div className="rounded-md border border-border bg-surface p-6 text-sm text-text">Panel derecho</div>

### Grid de cards · listas, galerías, catálogos

Usá `auto-fill` + `minmax` para que el grid se reacomode solo según el ancho disponible.

<Preview label="Card grid · auto-fill" background="muted" align="start" minHeight={260}>
  <div className="grid w-full grid-cols-[repeat(auto-fill,minmax(160px,1fr))] gap-4">
    {Array.from({ length: 8 }).map((_, i) => (
      <div
        key={i}
        className="h-24 rounded-md border border-border bg-surface p-4 text-sm text-text"
      >
        Card {i + 1}
      </div>
    ))}
  </div>
</Preview>

### Full-bleed · hero, banners

Un bloque que rompe el container y va de punta a punta. Útil para heroes y secciones de
marketing.

## Ritmo vertical

Dentro del main, mantené **3 ritmos** claros:

- Entre elementos del mismo bloque: `gap-4` (16px).
- Entre bloques relacionados: `gap-8` (32px).
- Entre secciones de página: `gap-16` o `gap-20` (64–80px).

Si mezclás ritmos sin jerarquía, la página pierde eje de lectura.

## Accesibilidad

- Orden lógico del DOM = orden visual. Si reordenás con `order` o `grid-row`, verificá
  que el tab-order siga siendo correcto.
- Skip-links (`a[href="#main"]`) obligatorios en layouts con sidebar largo.
- Landmarks semánticos: `<header>`, `<nav>`, `<main>`, `<aside>`, `<footer>` — no `<div>`
  con role manual si podés usar el elemento.
- Zoom al 200% no rompe layouts: evitá `width` fijos en `px` para containers — preferí `%`
  o `max-w`.

## Cuándo usar / Cuándo NO usar

<Callout variant="do">
  - Empezá con un patrón estándar (stack, sidebar, split). Solo salite si el contenido lo
    pide explícitamente.
  - Dejá que el grid se adapte con `auto-fill`/`auto-fit` antes que forzar columnas fijas.
</Callout>

<Callout variant="dont">
  - No mezcles `grid` y `position: absolute` en el mismo layout si no es estrictamente
    necesario. Son modelos mentales distintos y chocan.
  - No uses grillas de 12 cuando la data pide 3. Más columnas no es más flexibilidad —
    es más fricción.
</Callout>
